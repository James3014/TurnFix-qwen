# 釐清問題

「症狀/條件置信度不足」的判斷標準為何？

# 定位

Feature: 3. 功能需求（FR，最小集合）- 2) 自適應追問

# 多選題

| 選項 | 描述 |
|---|---|
| A | 模型輸出的機率分數低於某個閾值（如 0.7） |
| B | 映射到多個可能性相近的「症狀種子」 |
| C | 輸入的口語問題過於簡短或模糊 |
| D | 缺少必要的 Slot 資訊（如等級或地形） |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

這是「自適應追問」功能的核心觸發條件。一個明確的標準能確保系統在必要時才追問，避免不必要的打擾，直接影響使用者體驗。

# 優先級

High

---
# 解決記錄

- **回答**：A, B, C, D 組合 - 綜合考量模型機率分數、多個可能性相近的症狀種子、口語問題簡短模糊、缺少必要 Slot 資訊等因素。
- **更新的規格檔**：spec/features/自適應追問.feature
- **變更內容**：在自適應追問 Feature 中新增 Rule 和 Example，說明「症狀/條件置信度不足」的判斷標準是結合多個因素。
